# OUTLINE TEMPLATES CARNET

====

## 1. BLUF (Bottom Line Up Front)

| Descrittore | Contenuto |
|-------------|-----------|
| **Nome** | Bottom Line Up Front |
| **Descrizione** | Struttura che antepone la conclusione/raccomandazione principale, seguita da contesto, evidenze e azioni. Deriva dalla comunicazione militare NATO. |
| **Pubblico target** | Executive, decisori senior, contesti militari/governativi |
| **Tono comunicativo** | Diretto, assertivo, senza preamboli |
| **Quando usarlo** | Situazioni time-critical, briefing urgenti, quando il lettore ha poco tempo o vuole subito il "so what". Forza la chiarezza del messaggio. |
| **Quando evitarlo** | Analisi esplorative, temi controversi che richiedono costruzione graduale del consenso, audience che si aspetta argomentazione prima della conclusione. |
| **Lunghezza tipica** | Breve (1-2 pagine) |
| **Struttura chiave** | Bottom Line → Background → Key Facts → Implications → Recommended Action |

```markdown
# [Title]

## Bottom Line
[1-2 sentences: main conclusion/recommendation]

## Background
- [Essential context point 1]
- [Essential context point 2]

## Key Facts
- [Evidence 1]
- [Evidence 2]
- [Evidence 3]

## Implications
- [Consequence/impact 1]
- [Consequence/impact 2]

## Recommended Action
[What to do, who, by when]
```

## 2. Hypothesis-Driven

| Descrittore | Contenuto |
|-------------|-----------|
| **Nome** | Hypothesis-Driven |
| **Descrizione** | Struttura scientifica che parte da una tesi falsificabile, la sottopone a evidenze pro e contro, e conclude confermando o rivedendo l'ipotesi iniziale. |
| **Pubblico target** | Audience tecnici, stakeholder scettici, comitati di revisione |
| **Tono comunicativo** | Persuasivo, rigoroso, intellettualmente onesto |
| **Quando usarlo** | Raccomandazioni controverse, sfida allo status quo, strategie R&D, quando serve dimostrare rigore analitico e considerazione delle alternative. |
| **Quando evitarlo** | Decisioni operative urgenti, audience non interessata al processo logico, temi dove non esiste una tesi chiara da testare. |
| **Lunghezza tipica** | Media (3-6 pagine) |
| **Struttura chiave** | Hypothesis → Why This Matters → Supporting Evidence → Counter-Evidence/Risks → Conclusion → Implications & Next Steps |

```markdown
# [Title]

## Hypothesis
[Main thesis, falsifiable statement]

## Why This Matters
[Stakes, strategic relevance]

## Supporting Evidence

### Evidence 1: [Name]
- [Data/facts]
- [Source/reliability]

### Evidence 2: [Name]
- [Data/facts]
- [Source/reliability]

### Evidence 3: [Name]
- [Data/facts]
- [Source/reliability]

## Counter-Evidence / Risks
- [What could invalidate the thesis]
- [Critical assumptions]

## Conclusion
[Confirmation/revision of initial hypothesis]

## Implications & Next Steps
- [Action 1]
- [Action 2]
```

## 3. POR (Problem-Options-Recommendation)

| Descrittore | Contenuto |
|-------------|-----------|
| **Nome** | Problem-Options-Recommendation |
| **Descrizione** | Struttura decisionale che definisce un problema, presenta opzioni alternative con pro/contro, e conclude con una raccomandazione motivata. |
| **Pubblico target** | Policy maker, board, comitati decisionali |
| **Tono comunicativo** | Bilanciato, oggettivo, orientato alla scelta |
| **Quando usarlo** | Decisioni tra alternative discrete, policy brief, memo decisionali, quando serve mostrare che tutte le opzioni sono state valutate equamente. |
| **Quando evitarlo** | Situazioni senza reali alternative, analisi esplorative senza decisione imminente, quando una sola opzione è chiaramente dominante. |
| **Lunghezza tipica** | Media (3-5 pagine) |
| **Struttura chiave** | Problem Statement → Context → Options (A/B/C con Pro/Con/Feasibility) → Recommendation → Implementation Considerations |

```markdown
# [Title]

## Problem Statement
[Clear problem definition, 2-3 sentences]

## Context
- [Relevant factor 1]
- [Relevant factor 2]
- [Constraints]

## Options

### Option A: [Name]
- **Description:** [What it entails]
- **Pro:** [Advantages]
- **Con:** [Disadvantages]
- **Feasibility:** [High/Medium/Low]

### Option B: [Name]
- **Description:** [What it entails]
- **Pro:** [Advantages]
- **Con:** [Disadvantages]
- **Feasibility:** [High/Medium/Low]

### Option C: [Name]
- **Description:** [What it entails]
- **Pro:** [Advantages]
- **Con:** [Disadvantages]
- **Feasibility:** [High/Medium/Low]

## Recommendation
[Recommended option + rationale in 2-3 sentences]

## Implementation Considerations
- [Resources required]
- [Timeline]
- [Risks to mitigate]
```

## 4. Minto-Custom

| Descrittore | Contenuto |
|-------------|-----------|
| **Nome** | Minto-Custom |
| **Descrizione** | Adattamento della Piramide di Minto: parte dalla conclusione, introduce contesto e complicazione, poi sviluppa argomentazioni gerarchiche (key lines → arguments → evidence). |
| **Pubblico target** | Tutti i professionisti, audience miste, contesti consulenziali |
| **Tono comunicativo** | Strutturato, professionale, scalabile |
| **Quando usarlo** | Analisi strategiche complete, report consulenziali, documenti che devono funzionare sia per lettura rapida che approfondita. Massima flessibilità di scala. |
| **Quando evitarlo** | Comunicazioni molto brevi dove la struttura risulterebbe sovradimensionata, contesti informali, brainstorming non ancora maturi. |
| **Lunghezza tipica** | Flessibile (2-20+ pagine) |
| **Struttura chiave** | Main Conclusion → Introduction (State of the Art + Complication) → Key Lines (con Arguments e Supporting Evidence) → Implications → Next Steps |

```markdown
# [Title]

## Main conclusion
[The key question this analysis addresses + Main conclusion/recommendation]

## Introduction

### State of the Art
[Stable context the reader already knows or accepts]

### Complication
[What changed, what's at stake, the tension]


## Key Line

### Argument 1: [Key point]
- [Supporting evidence]
- [Supporting evidence]
- [Supporting evidence]

### Argument 2: [Key point]
- [Supporting evidence]
- [Supporting evidence]
- [Supporting evidence]

### Argument 3: [Key point]
- [Supporting evidence]
- [Supporting evidence]
- [Supporting evidence]

## Implications
- [What this means for the reader]
- [Strategic consequences]

## Next Steps / Recommendations
- [Action 1]
- [Action 2]
```

## Selection Guide

| Template | Use When | Audience | Tone |
|----------|----------|----------|------|
| BLUF | Time-critical, action-oriented | Executives, military | Direct |
| Hypothesis-Driven | Proposing non-obvious conclusions | Technical, skeptical | Persuasive |
| POR | Decision required among alternatives | Policy makers, boards | Balanced |
| Minto-Custom (^) | Comprehensive strategic analysis | All professional | Structured |

(^) **Default_Template**: Minto-Custom with 3 Key Lines, with 3 Arguments each, with 3 bullet points each
